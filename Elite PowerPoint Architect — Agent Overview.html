Agent Memories
10 persistent rules — Click to expand
01Option Explicit + Dim + Set
Every module begins with Option Explicit. Every variable declared with Dim. Every object uses Set.....
These three rules prevent 80% of all VBA errors. Option Explicit forces declaration. Dim types every variable. Set assigns objects correctly.....
02Late Binding — As Object Always
ALL PowerPoint objects declared As Object. Never As Presentation, As Slide, As Shape. ByVal for all parameters.....
Prevents ByRef argument type mismatch errors across different PowerPoint versions. Universal compatibility.....
03ppLayoutBlank (12) for All Slides
Always use blank layout, build everything from scratch with shapes. Never use placeholder-based layouts.....
Placeholder-based layouts vary across systems and cause errors. Building from scratch ensures consistency.....
04PageSetup — Points Only
16:9 = 960×540 points. 4:3 = 720×540 points. Never pass enumeration values to PageSetup.....
Passing enumerations causes "Invalid enumeration value" errors. Always use direct numeric point values.....
05Forbidden Methods — Do Not Exist
CreateBasicShape, AddBasicShape, MakeShape, CreateSlide — NONE of these exist in the PPT VBA object model.....
Correct methods: Shapes.AddShape(), Shapes.AddTextbox(), Shapes.AddLine(), Shapes.AddPicture(), Slides.Add().....
06Safe Animation Effect IDs
Fade=10, Appear=1, FlyIn=2, Float=42, Wipe=53, Zoom=55. Triggers: OnClick=1, AfterPrevious=3.....
Only use verified safe values. Unknown animation IDs can cause runtime errors.....
07Safe Transition Values
Fade=3844, Push=3334, Wipe=3335, Cut=257, None=0. Default to Fade if unsure.....
Always verify transition values before using. Fade (3844) is the universal safe default.....
08Headline Story Test
Read ONLY slide titles in sequence — they must tell the complete story independently.....
If someone reads just the headlines and understands the full narrative, the structure is correct. Revise until passing.....
09Modular Architecture
One Sub per slide + master orchestrator. For 25+ slides, output in batches. Max ~2500 lines per response.....
Each CreateSlideN sub is self-contained and independently testable. Prevents timeout and enables debugging.....
10Error Handling in Every Sub
On Error GoTo ErrHandler + Exit Sub + Debug.Print with Err.Number and Err.Description.....
Catches runtime errors gracefully instead of crashing. Every single Sub must include this pattern.....
