Agent Memories
18 persistent rules — Click to expand
01Accuracy Over Speed
Primary purpose is quick, accurate, precise SQL queries. Accuracy more important than speed.....
Never sacrifice correctness for faster responses. Precision is the core mandate.....
02Three Campaign Types Only
Support exactly: Email Engagement, WhatsApp Single Send, Rep to Digital. All templates in Datasets.....
Retrieve and use only these templates from Datasets. No other campaign types supported.....
03EE Flat File Naming
{BU}_P_EE_{BRAND}_{SENDOUTDATE}_FlatFile — e.g., AR_P_EE_TRELEGY_05062025_FlatFile.....
Strict naming convention. BU code + P + EE + Brand (uppercase) + Date (DDMMYYYY) + FlatFile.....
04DE Naming (No FlatFile Suffix)
Data Extension follows same pattern without _FlatFile suffix.....
Example: AR_P_EE_TRELEGY_05062025 or AR_P_WA_TRELEGY_05062025.....
05EE Segmentation Options
Three ways: Specialty values, Flat File join (Veeva ID/MDMID), or entire Optin DE.....
Each segmentation path produces different query structures. Must ask user which one.....
06WA Optout Conditional Queries
EncodedMobileNumber + OptOutSuffix only if template has optout link.....
Always ask user about optout link before generating queries. Include or exclude based on answer.....
07R2D Sub-type Query Matrix
Sub-type 1: RepEmailCondition. Sub-type 2: ProductMetrics trio. Sub-type 3: All.....
Sub-type determines which conditional queries to include/exclude from output.....
08R2D CalculateDayOf Logic
Number of emails → number of CalculateDayOf queries. Max 4 emails = 4 queries.....
1 email = 1 query, 2 = 2, 3 = 3, 4 = 4. Only included when Day Exclusion = Yes.....
09Day Addition Calculation
Base days + excluded day count. E.g., 2 days + 3 excluded = 5 in DATEADD.....
For each excluded day, calculate days to add to reach next valid non-excluded day. Dynamic CASE/WHEN.....
10DATEDIFF Always Negative
User provides 2 → becomes -2 in ProfilesEmailsCallDetailsSlidesJoin query.....
Sign conversion is mandatory. Never use positive value in the DATEDIFF clause.....
11Brand UPPERCASE Mandatory
Brand names must always be converted to UPPERCASE before insertion.....
Send Out Date must be DDMMYYYY with exactly 8 digits. Both validated before generation.....
12Never Generate From Own Knowledge
Every query must come directly from Datasets. If template not found, inform user.....
Zero tolerance for invented queries. Dataset is the single source of truth.....
13CurrentRating Optional
AND (P.CurrentRating IN ('A','B','',' ') OR P.CurrentRating IS NULL) — only if user requests.....
Never added by default. Must explicitly ask user. Include only on confirmation.....
14R2D Sub-type 4 Flat File Option
Optional mass email on behalf of Rep using Flat File. FF name: {BU}_P_R2D_{BRAND}_{DATE}_FF.....
Only for Sub-type 4. Join on VeevaID or MDMID. Adds condition to ProfilesEmailsCallDetailsSlidesJoin.....
15Dynamic Day Exclusion Calc
For each excluded day, calculate days to add to reach next valid day. Only excluded days get WHEN clauses.....
General rule: identify excluded days, find next valid day, calculate offset for each. Non-excluded days pass through unchanged.....
16R2D Sub-type 4 Flat File Naming
{BU}_P_R2D_{BRAND}_{SENDOUTDATE}_FF. Join on VeevaID or MDMID.....
Example: AR_P_R2D_AREXVY_02092025_FF. Only applicable for Sub-type 4 with Flat File join.....
17Slide Duration Optional
{SLIDE_DURATION_MIN} is optional. If No, remove entire condition block. Not a missing placeholder.....
Never flag removed optional section as error. Never trigger REDO_STEP for intentional removal.....
18Automation Config Rules
Automation name = Target DE name. EE/WA: 1st=Overwrite, rest=Update. R2D: ALL=Update.....
Always display Target DE and Query Action for each query. Automation name must match Target DE.....
