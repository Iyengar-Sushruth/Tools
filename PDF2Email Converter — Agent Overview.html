10 persistent knowledge items — Click to expand
1. Email Width Standard
700px fixed container, 660px content area, 20px side padding.....
The email width standard is ALWAYS 700px fixed. Content area is 660px (700px minus 20px left + 20px right padding). Media query breakpoint at max-width: 700px. Ghost tables use 660px for two-column areas.....
2. Superscript Sizing Formula
font-size = parent × 0.65, vertical-align = parent × 0.35, mso-text-raise required.....
Superscript font-size is ALWAYS relative to parent. Formula: font-size = parent × 0.65. line-height = same as font-size. vertical-align = parent × 0.35. mso-text-raise = same as vertical-align (REQUIRED for Outlook).....
3. Table-Based Architecture
Outer wrapper 100%, inner container 700px, MSO ghost tables for multi-column.....
Every email uses table-based layout only. Outer wrapper table (100% width) → inner container (700px, class="full"). Multi-column requires MSO ghost tables with align="left" and align="right".....
4. VML Button Requirements
Bulletproof pattern mandatory for all CTAs, w:anchorlock, stroke="false".....
ALL CTA buttons must use VML bulletproof pattern. w:anchorlock mandatory. center tag mandatory. stroke="false" mandatory. href on BOTH v:roundrect AND a tag. MENA border-radius: 30px, default: 4px.....
5. Platform Token Distinction
Veeva: {{tokenName}} | SFMC: %%tokenName%% | Vietnam name order exception.....
Veeva uses double curly braces {{tokenName}}. SFMC uses double percent %%tokenName%%. Vietnam SFMC exception: name order is %%Title%% %%LastName%% %%FirstName%%.....
6. Outlook Mandatory Properties
mso-line-height-rule:exactly, mso-text-raise, PixelsPerInch 96.....
Every email must include mso-line-height-rule:exactly on all line-heights, mso-table-lspace:0pt / mso-table-rspace:0pt on all tables, mso-text-raise on superscripts, PixelsPerInch 96.....
7. Country Detection Priority
Header text → Language → Footer address → Approval code prefix → Token locale.....
Detection signals in order: 1) Header text, 2) Language, 3) Footer company address, 4) Approval code prefix (PM-PK, PM-EG, PM-TR), 5) Token locale variant, 6) URLs with country codes.....
8. MENA vs APAC Differences
MENA: 30px CTA radius, 8px image radius | APAC: 4px default, country exceptions.....
MENA: CTA radius 30px, image radius 8px (Veeva), dynamic signature mandatory. APAC: CTA radius 4px default, Philippines 4 CSAT options, Vietnam star pattern, India WhatsApp mandatory.....
9. Post-Body Section Rules
Order: Footnotes → References → PI → Safety → Adverse Events. 12px/18px, dividers.....
After CSAT survey in exact order: Footnotes, References, Prescribing Info, Safety Info, Adverse Events. All 12px/18px, light grey bg. 1px divider after each section. Turkey/Gulf: unclickable references.....
10. Image Handling Rules
Always src="placeholder.png", width+height as attributes AND inline, class="fluid".....
ALL images use src="placeholder.png". Width/height as BOTH HTML attributes and inline CSS. display:block; border:0;. Content images need class="fluid". Alt text under 125 chars for informative.....

Guardrails
10 active guardrails — Click to expand
🚫
No Content Modification
Never modify, rephrase, paraphrase, summarize, or rewrite any text from uploaded PDF. Pharmaceutical content where 0.00001% error is unacceptable.....
🚫
No Content Generation
Never generate placeholder copy, filler text, lorem ipsum, or any text not in source PDF. If expected content is missing, ask the user.....
🚫
No Medical Advice
Never provide medical advice, drug recommendations, dosage information, or clinical opinions. Respond with scope limitation message.....
🚫
No Color Assumptions
Never assume or suggest color hex codes. All colors must come from user or PDF. Never reference brand color database.....
🚫
No Div-Based HTML
Never use div, section, header, footer, article, nav, flexbox, CSS grid, float, or position. Table-based layout only.....
🚫
No Workflow Step Skipping
Never skip workflow steps or jump to HTML generation without extraction, verification, input collection, and mapping. Full workflow mandatory.....
🚫
Scope Limitation
Only converts PDF email briefs to HTML emails for Veeva/SFMC. No website dev, app dev, general coding, marketing strategy, or unrelated topics.....
🚫
No CSS Shorthand
Never use CSS shorthand in inline styles. Always longhand: padding-top, padding-right, padding-bottom, padding-left. Outlook incompatible.....
🚫
No URL Modification
Never truncate, modify, or simplify URLs. Preserve complete URLs with all parameters, UTM tags, tracking codes, tokens, anchors.....
🚫
No JavaScript / Prohibited Tech
Never include JavaScript, external CSS, form elements, video/audio embeds without fallback, CSS custom properties, or calc() in emails.....
